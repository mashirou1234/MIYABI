# C2 3D基盤 最小契約

最終更新: 2026-03-07

## 1. 目的

C2 では、既存 2D ランタイムを壊さずに 3D 描画の最小導入範囲を固定する。
本書は「何を入れるか」だけでなく、「何をまだ入れないか」と「2D 回帰条件」を同時に定義する。

## 2. C2 で導入する最小機能

### 2.1 描画パス

- 3D パスは透視投影を前提とし、深度テストを有効化する。
- 2D スプライト / テキスト / UI は従来どおり画面座標系で描画し、3D パスの後段オーバーレイとして扱う。
- 初手では 3D と 2D を同一フレーム内で共存させるが、ポストプロセスや複数カメラは対象外とする。

### 2.2 カメラ契約

- 最小カメラは 1 台のみ。
- 必須パラメータは `fov_y` / `near` / `far` / `position` / `look_at`。
- C2 の初手では editor camera は作らず、ゲームまたはサンプルが指定する単一ビューのみを扱う。

### 2.3 メッシュ契約

- 最小メッシュ入力は静的メッシュのみとする。
- 初手の外部フォーマットは `OBJ` 1 系統に固定する。
- スキニング、ボーン、ブレンドシェイプは C2 対象外とする。
- 2D と衝突しないよう、3D 用メッシュ登録は 2D の `mesh_id=1` 運用から独立した ID 空間で扱う。

### 2.4 ライト契約

- 最小ライトは directional light 1 灯のみ。
- 影生成、IBL、ポイントライト、スポットライトは C2 対象外とする。
- シェーディングは「陰影が分かる」ことを優先し、品質は Lambert 相当でよい。

### 2.5 深度とマテリアル

- 深度書き込み / 深度比較は 3D パスのみで有効化する。
- 2D パスでは従来どおり描画順優先とし、深度バッファを前提にしない。
- C2 のマテリアルは「単色 + テクスチャ + directional light」の最小セットに限定する。

## 3. 2D 互換条件

3D 導入中も、以下を満たさない変更は C2 着手として受け入れない。

- `build/core/miyabi` の既存 2D 起動経路がそのまま残る。
- `logic` の 2D 仕様 (`Title -> InGame -> Pause -> Result`) が壊れない。
- 2D のスプライト、テキスト、UI ボタンが 3D 用深度設定の影響を受けない。
- 配布 ZIP 同梱物に 3D 用アセットを追加しても、既存 2D 配布スモークを破壊しない。
- `core -> sample_game` の依存逆流を新設しない。

## 4. C2 の最小回帰コマンド

- `cargo test --manifest-path logic/Cargo.toml --lib -- --nocapture`
- `./scripts/test_game_track_g2.sh`
- `./scripts/test_distribution_smoke.sh`
- 必要に応じて `cmake --build build -j`

## 5. C2 の初手実装タスク

本書から直ちに切り出す実装タスクは次の 4 件を最小単位とする。

1. `C2-01`: 透視カメラ + 3D パス分離
   - ビュー / 射影行列の導入
   - 3D パスだけ深度テストを有効化
2. `C2-02`: OBJ メッシュ登録とロード導線
   - `mesh_id` 登録 API
   - 最小 1 メッシュのロード確認
3. `C2-03`: directional light 1 灯と最小陰影
   - unlit 2D パスと混線しない shader 契約
4. `C2-04`: 2D/3D 共存回帰ハーネス
   - 2D スモーク + 3D 最小描画ログを同時に採取

## 6. C2 で意図的に後回しにする項目

- スキニング / アニメーション
- 複数ライト
- シャドウ
- ポストプロセス
- シーンエディタ
- 3D 向けアセット変換パイプラインの高度化

C2 の完了判定は「3D を入れ始められること」であり、「3D 制作が快適であること」ではない。後者は C3 以降で扱う。
