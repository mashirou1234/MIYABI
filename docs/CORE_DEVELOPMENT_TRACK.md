## 1. 目的

エンジンとしての基盤機能を段階的に完成させ、2D/3D開発を継続可能にする。

## 2. ステージ定義

### C0: Core Runtime（現状）

- FFI経由でゲーム更新/描画データ受け渡しが成立
- 2D描画/入力/音声/2D物理の最小統合が成立
- SDK v0.1 配布物が成立
- Core Save サブシステム定義が成立（未接続）

### C1: 2D Engine Baseline

- 2D制作向けの実運用機能を提供
- アセットパイプライン、テスト/CI、性能回帰監視を導入
- 2D開発が継続可能な開発基盤を成立

### C2: 3D Foundation

- 透視投影、3Dメッシュ、ライト、深度処理の最小導入
- 2D機能を破壊せず 3D基盤を共存させる

### C3: 3D Engine Baseline

- 3D制作向けの実運用機能を提供
- 表現拡張、性能最適化、3D向けツールを整備

### C4: Ecosystem Competitiveness（長期）

- 開発体験（デバッグ/拡張/ドキュメント）を整備
- マルチプラットフォーム運用と互換性方針を確立

## 5. ステージ別 完了定義（DoD）

各ステージは次の共通フォーマットで判定する。

- 完了定義:
  - 対象機能がステージ定義どおりに実装済み
  - 既存機能を壊さない最小回帰確認が完了
  - 関連ドキュメントが更新済み
- 判定エビデンス:
  - 実行ログ、テスト結果、またはスクリーンショット
  - 更新したドキュメント差分

### C0 完了定義

- 完了定義:
  - FFI経由の更新/描画データ受け渡しが動作する
  - 2D描画/入力/音声/2D物理の最小統合が動作する
  - SDK v0.1 配布物が再生成可能な状態である
- 判定エビデンス:
  - 最小起動確認ログ（ビルド・起動）
  - SDK成果物生成ログ

### C1 完了定義

- 完了定義:
  - 2D制作向け実運用機能が定義どおり提供される
  - アセットパイプライン、CI、性能回帰監視が運用可能
  - 2D開発継続に必要な手順がドキュメント化されている
- 判定エビデンス:
  - 2D機能の最小再現手順ログ
  - CI/性能チェックの通過記録

### C2 完了定義

- 完了定義:
  - 透視投影、3Dメッシュ、ライト、深度処理が最小動作する
  - 2D既存機能に互換性破壊がない
  - 3D基盤導入の境界条件が文書化されている
- 判定エビデンス:
  - 3D描画の最小動作確認ログ
  - 2D回帰テストの通過記録

### C3 完了定義

- 完了定義:
  - 3D制作向け実運用機能が定義どおり提供される
  - 表現拡張と性能最適化の基準を満たす
  - 3D向けツール運用手順が整備されている
- 判定エビデンス:
  - 3D運用シナリオの最小再現ログ
  - 性能計測結果と基準比較

### C4 完了定義

- 完了定義:
  - デバッグ/拡張/ドキュメントの開発体験が運用可能
  - マルチプラットフォーム運用方針が適用済み
  - 互換性方針と更新ルールが継続運用可能
- 判定エビデンス:
  - プラットフォーム別確認結果
  - 互換性方針に基づく運用記録
