# C3/C4 3D制作基盤と競争力評価軸

最終更新: 2026-03-08

## 1. 目的

C3 では「3D タイトルを継続制作できる基盤」を、C4 では「他エンジン比較で弱点を議論できる評価軸」を固定する。
本書は実装一覧ではなく、中長期バックログを切るための分類表として使う。

## 2. C3 必須要素一覧

### 2.1 ランタイム / 描画

- マテリアル拡張（複数ライト、基本陰影の切替）
- 3D アニメーション最小再生（少なくとも clip 再生）
- シーンロード / アンロード
- カリングまたは同等の負荷抑制
- 代表 3D シーンの性能ベースライン

### 2.2 アセット / 制作導線

- メッシュ / テクスチャ / マテリアルの検証手順
- シーンデータの保存形式と更新ルール
- 3D サンプルシーン 1 本以上
- 配布物へ 3D アセットを同梱する手順

### 2.3 ツール / 運用

- レベル / シーン編集を補助する最小ツール
- アセット不整合を検知する診断コマンド
- 3D 回帰テスト手順
- 既知制約を追跡するドキュメント導線

## 3. C4 競争力評価軸

| 評価軸 | 何を見るか | 最低限の問い |
| --- | --- | --- |
| 初期導入速度 | 新規開発者が 1 日以内に 3D サンプルを動かせるか | README とテンプレだけで起動できるか |
| 反復速度 | アセット差し替えやシーン更新の速さ | 1 変更あたりの再ビルド / 再読込手順が短いか |
| デバッグ容易性 | 問題発見と原因切り分けの速さ | 描画 / 物理 / アセットの診断手段があるか |
| 拡張性 | 機能追加時の依存方向の健全性 | `core -> logic -> sample_game` の境界を壊さず拡張できるか |
| 運用性 | 配布、CI、性能監視の継続性 | 3D でも配布スモークと性能回帰を回せるか |
| ドキュメント成熟度 | 設計と運用の正本が揃っているか | 実装前に「何を作るか」が文章で判断できるか |

### 3.1 評価テンプレート運用

- 正本テンプレート: `docs/templates/CORE_3D_DEVEX_EVALUATION_TEMPLATE.md`
- 初回試験記入: `docs/reviews/CORE_3D_DEVEX_EVALUATION_2026-03-08.md`
- 利用タイミング:
  - C4 直結 Issue / PR を閉じる前
  - C4 follow-up Issue を切り出す前
  - 主要 3D 導線に大きな変更が入った直後
- 最小ルール:
  - 6 評価軸すべてを同じ表で埋める
  - 各行に証跡リンクを最低 1 件残す
  - `score <= 1` の項目は、次の Issue 候補を必ず書く

## 4. 中長期バックログの分類

### 4.1 近接バックログ（C3 直結）

- 3D シーン資産のロード / 保存
- 3D 性能ベースラインの自動採取
- 3D アセット検証と診断コマンド
- 3D サンプルシーンの CI スモーク

### 4.2 拡張バックログ（C4 直結）

- プラグイン / モジュール方針
- デバッグ可視化オーバーレイ
- マルチプラットフォーム検証導線
- チュートリアル / サンプル群の体系化

## 5. follow-up Issue 候補

次に着手する follow-up Issue は少なくとも次の 6 件とする。2026-03-08 時点で 1-6 は実装済みであり、以後は `4.2` の C4 直結バックログから再分解して継続する。

1. `[Core][C2] 透視カメラと 3D パス分離を実装する`（`#357`, 2026-03-08 実装済み）
2. `[Core][C2] OBJ メッシュロードと mesh registry を導入する`（`#358`, 2026-03-08 実装済み）
3. `[Game][G4] 3D arena 最小シーンを起動可能にする`（`#359`, 2026-03-08 実装済み）
4. `[Core][C3] 3D シーン性能ベースライン採取を自動化する`（`#366`, 2026-03-08 実装済み）

6. `[Core][C4] 3D 開発体験の評価テンプレートを整備する`（`#368`, 2026-03-08 実装済み）

## 6. 今回あえて固定しないもの

- 具体的な外部 DCC ツールの選定
- シェーダ言語の高度な抽象化方針
- 商用エンジンとの定量ベンチマーク値
- マーケット調査や事業計画

C3/C4 では「全部入り」を目指さず、3D 制作継続に必要な最低限の基盤と、その不足を説明できる評価軸を優先する。
